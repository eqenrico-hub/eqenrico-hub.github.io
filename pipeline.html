Gender Classification Pipeline — Transparencia.be FOI Dataset

5-Tier Cascade: Tier 0 Greeting → Larousse → Statbel → WGND 2.0 → Qwen 2.5 LLM (Tiers 0–3 deterministic, Tier 4 near-deterministic)
Enrico Quaglia -March 2026

This pipeline classifies requester and respondent gender using a five-tier cascade. Tier 0 (response greeting) takes precedence for requesters because it reflects how the administration perceived the requester’s gender — the signal that shapes their responsiveness, which is what this study examines. Tiers 1–3 use deterministic dictionary lookups as a proxy when no greeting is available. Tier 4 (Qwen 2.5 LLM) is a probabilistic fallback for unresolved cases.

Annotation Notes

▶ 1 Closing formula
French administrative letters conventionally end with a politeness formula (e.g., "Cordialement", "Veuillez agréer mes salutations distinguées") followed by the signer's name. The pipeline searches the last 500 characters for such formulas and extracts everything from the formula to the end of the text -this is the "signature zone" where the author's name appears.

▶ 2 Why 308 requester cases for Ollama
308 cases remain after Tier 0 cross-checking resolves 71 of the original 379 unmatched cases. These 308 include 158 cases where a name token was extracted but not found in any of the three dictionaries (possibly a surname, rare name, or false positive), plus 150 cases where no name was extracted at all. Ollama receives only the signature zone text (not the entire document) to minimize the risk of picking up names from the body of the letter that belong to other people. Since the FOI data could not be sent to cloud-based LLMs for privacy reasons (Transparencia.be requested that data not be processed by external services), a locally hosted model (Qwen 2.5 via Ollama) was used. Qwen 2.5:7b was selected after an initial run with Llama 3.2 (3B parameters) revealed a high error rate upon manual review. The model runs entirely on the researcher’s machine with no data leaving the local environment.

▶ 3 Why respondent Ollama success is higher (9.9% vs 2.9%)
The 1,237 respondent cases sent to Ollama are those where the heuristic and spaCy found NO name at all in the reply text. These are predominantly institutional signatures ("Le Secrétariat communal", "Service Urbanisme"), automated acknowledgments, or email disclaimers -texts that genuinely contain no personal first name. With Qwen 2.5, the relationship inverts compared to the earlier Llama 3.2 run: the respondent M/F rate (9.9%) is now higher than the requester rate (2.9%), because the improved model is better at finding personal names in institutional reply texts that are longer and more varied than citizen request signatures.

▶ 4 BOTH / UNKNOWN (UNISEX)
Names like Claude, Dominique, or Camille are used for both sexes in French. Larousse marks these as "mixte". For Statbel, a conservative 90% threshold was applied when building the reference database: a name is classified M or F only when ≥90% of bearers in Belgian civil registry data are of that sex, ensuring high confidence. Names below this threshold are classified as BOTH (unisex) rather than M/F -they are never counted among the gendered results.

Limitation: In theory, a small number of UNKNOWN cases from the pipeline could be names that a less conservative threshold might have resolved. However, this concerns at most a handful of cases: Tier 2 (Statbel) classifies only 52 requester and 40 respondent cases in total -a marginal share of the overall sample, where the vast majority are already identified by Tier 1 (Larousse) and Tier 3 (WGND). The practical impact of this threshold on the final UNKNOWN count is therefore negligible.

Future integration: The 135 requester and 24 respondent BOTH cases could potentially be partially resolved through probabilistic gender assignment -a standard technique in gender studies (Blevins & Mullen, 2015; Lax-Martinez et al., 2021). Rather than discarding unisex names, each case would be weighted by the actual male/female frequency ratio in population data (e.g., if Statbel shows 62% of “Dominique” bearers in Belgium are female, the case would count as 0.62 F + 0.38 M in the statistical analysis). This was not implemented in the current pipeline, which uses a conservative binary approach, but could be explored in future iterations to recover some of the BOTH cases without sacrificing confidence.
▶ 5 Hallucination filter
This filter applies only to cases where the LLM returned a name with a gender classification (M/F or flagged for REVIEW). After the LLM returns a name, the pipeline checks that the name actually appears in the signature zone text it was given. If the name is absent (the LLM invented it), the classification is discarded and reclassified as UNKNOWN. Cases where the LLM returned UNKNOWN or produced no result are not affected -there was no name claim to verify.
▶ 6 Why only 146 cases need review
Tiers 1–3 are deterministic dictionary lookups — same input always gives same output, no review needed. Only Tier 4 (LLM) is probabilistic, so ALL 132 of its M/F outputs (9 requester + 123 respondent) are flagged for human verification. Of these, 17 were corrected (13% error rate): 9 cases where the LLM should have returned UNKNOWN (hallucinations, initials-only, unisex names), 6 wrong gender assignments, and 2 encoding/format issues. Additionally, 14 Tier 0 disagreements (where the response greeting diverges from dictionary classification) were manually reviewed — breakdown: 3 group requests (first co-signer extracted but admin addressed a different one), 11 dictionary overrides. In all 14, the greeting correctly reflected perceived gender; no adjustments needed. (9 former disagreements were eliminated by improved multi-line neutral greeting detection.) Total: 146 cases reviewed. UNKNOWN cases validated separately: 259 from Llama run (96.5% genuinely unclassifiable), 200-sample from Qwen run (100% genuinely unclassifiable).
▶ 7 How platform boilerplate is identified
The Transparencia.be platform appends boilerplate to every message. The pipeline uses a two-stage cleaning system:

Outgoing texts (citizen requests): 4 prefix markers (dashes, Merci d’utiliser, Gelieve dit, Please use this email).
Incoming texts (official responses): A first-stage cut using 9 markers (quoted-message headers, email forwarding headers, and 3 shared platform markers), followed by a second-stage pass using 8 additional platform footer strings plus quoted-message patterns and “References / Visible links” sections.

Everything from the first marker onward is stripped before signature zone extraction. For respondent texts, the pipeline also strips quoted original requests (detected via “-----Message d’origine-----”, “Le [date] a écrit:”, and similar email quoting conventions) to prevent the requester’s signature from being mistaken for the respondent’s.

D -Key Differences from Previous Methodology

Previous Classifications vs. New Pipeline

The article draft (BE_FOI_draft_13032026.docx) uses previous gender classifications. Below are the key differences with the new deterministic pipeline.

Pipeline Evolution

The pipeline evolved through two iterations. An initial run with Llama 3.2 (3B) produced a 74% error rate in Tier 4 M/F classifications (122/164 cases incorrect, identified by co-author review). The final pipeline uses Qwen 2.5 (7.6B) with Tier 0 greeting detection, which resolved this issue: a 200-sample validation of Qwen UNKNOWN cases found 100% genuinely unclassifiable, and the model produces higher agreement with ChatGPT baselines across all metrics.

1. Respondent Sex Composition -Reversed

This is the single biggest change.

Likely cause: the previous method may have inferred sex from greeting formulas like “Madame” (which address the requester, not the respondent), while the pipeline strictly uses the signature at the end of the text.

Impact: The sentence “58.19% of responders are identified as female” (current Table 2) must be reversed. The chi-square test for gender-neutral assignment will likely become significant.

4. Robust Findings (Confirmed)

  • T2 (Urban planning): Female respondents boost success rates by ~18pp regardless of requester sex. Holds in both methods (Previous: +17–20pp, Pipeline: +18pp). Most robust finding in the dataset.
  • T4 (Council deliberations): Male requesters massively outperform female requesters. Not only confirmed but amplified (Previous: 25–28pp gap → Pipeline: 56–58pp gap).
  • H1 (requester gender & responsiveness): Male requesters more likely to receive a response. Should hold with new classifications.
  • Requester agreement: 95.9% of M confirmed, 98.7% of F confirmed -requester classifications barely changed (gender flips <1%).

5. What Must Change in the Article

  • Table 2 (gender combo cross-tabulation): all percentages change
  • Descriptive findings (gender interactions section): respondent composition reversal, new combo success rates
  • Retention balance analysis: percentages change (currently “54.2% of male requesters” and “58.8% of female requesters”)
  • Regressions (Models IV–VI): need re-running with new gender variables
  • H4 (same-gender moderation): effect weakens but direction holds

6. What Does Not Change

  • All non-gender variables (topics, sentiment, request status) are identical -same dataset
  • Requester gender classifications: 95.9% M confirmed, 98.7% F confirmed (flips <1%)
  • H1, H2, H3 findings: depend on requester gender, which barely changed
  • Structural patterns (T2 respondent effect, T4 requester gap) are robust across both methods
