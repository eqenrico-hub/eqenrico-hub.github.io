Folder Structure
Each FOI request is stored as a folder containing outgoing (citizen → administration) and incoming (administration → citizen) text files.

Rule: The pipeline always selects the first outgoing file (requester) and the first incoming file (respondent) per request -chronologically earliest timestamp. This ensures we identify the original signers, not follow-up authors.

Step 5 of 17
Boilerplate Removal
Transparencia.be appends a platform footer to every message. The pipeline detects and strips it so it doesn't interfere with name extraction.
Monsieur, Par la présente, je sollicite l'accès aux documents relatifs au permis d'urbanisme délivré le 14 mars 2022 pour la parcelle située rue de la Loi 42, 1000 Bruxelles. Je vous remercie d'avance pour votre réponse. Cordialement, Jean Dupont --- Si vous trouvez ce service utile en tant que plateforme de transparence, vous pouvez soutenir notre travail en faisant un don sur transparencia.be/soutenir
The pipeline recognizes French, Dutch, and English boilerplate markers: “Si vous trouvez ce service utile”, “Gelieve dit e-mailadres te gebruiken”, “Please use this email address”. Everything from the first marker onward is removed.

Step 7 of 17
Name Extraction
Two methods scan the signature zone in parallel to find the signer's name.
Signature Zone
Cordialement,
Jean Dupont
Method A: spaCy NER
spaCy is an open-source Natural Language Processing library (Honnibal & Montani, 2020). It uses a pre-trained French language model (fr_core_news_sm) that was trained on thousands of annotated French texts to recognize named entities: people, places, organizations, etc. When it reads text, it labels each word span with a category. We use the PER (person) label to find names.

Found: “Jean Dupont” - PER
✅
Method B: Heuristic (rule-based)
A simpler, non-AI approach: scan each line in the signature zone for 2 to 4 capitalized words where at least one matches a known first name in our 3.49M name database. This catches names that spaCy might miss, and works without any machine learning.

Found: “Jean Dupont” - match
✅
Both methods agree: “Jean Dupont” → name extraction confirmed. When methods disagree: first, only names matching a known first name in the database are considered valid. If both are valid, the name appearing earlier in the signature zone is preferred (first signer = most relevant identity). If only one method finds a name, that result is used.

Isolating the First Name
Step 7 gave us the full name (e.g., "Massenaux Philippe"). But which part is the first name and which is the surname? In Belgian admin practice, signers write either "Jean Dupont" (first name first) or "Dupont Jean" (surname first). We need to figure out which token is the first name before we can look up its gender.

Result: first_name = “Philippe” → normalized to “PHILIPPE” (strip accents, uppercase). Gender is not determined yet - that's the next step.
This works regardless of name order. "Jean Dupont" → "Jean" matches first. "Massenaux Philippe" → "Massenaux" fails, "Philippe" matches. Short words like "de", "du", "le" (under 3 characters) are automatically skipped by a minimum length filter. Longer connectors like "van" or "den" simply don't appear in the first name databases, so they fail the lookup naturally. Compound names like "Jean-Pierre" are tried as-is first, then just "Jean".
Step 9 of 17
Gender Lookup - The Tier Cascade
Now we know the first name (e.g., "PHILIPPE"). To find its gender, we run a five-tier cascade. First, we check how the administration addressed the requester in their response (Tier 0). If a gendered greeting is found, that takes precedence as direct evidence of perceived gender. Otherwise, we check the three name databases in priority order (Tiers 1–3). If no match is found, the LLM attempts classification (Tier 4). The order matters because sources can disagree on ambiguous names — Tier 0 (response greeting) takes precedence as it reflects the administration’s direct perception. Among name-based tiers, Larousse (the most authoritative for French names) has priority.
Tier 0: Response Greeting Patterns
The pipeline scans the first line only of the administration’s response for these patterns:
Female: “Chère Madame/Mme/Mademoiselle”, “Madame [Name],”, “Madame,”, “Bonjour/Bonsoir Madame”, “Geachte Mevrouw” (NL), “Beste Mevrouw” (NL)
Male: “Cher Monsieur/Mr”, “Monsieur [Name],”, “Monsieur,”, “Bonjour/Bonsoir Monsieur”, “Geachte Heer” (NL), “Beste Heer” (NL)
Skipped (neutral): “Madame, Monsieur”, “Madame, Messieurs”, “Mesdames, Messieurs”, “Chère Madame, Cher Monsieur” — these address both genders, so no single perceived gender can be inferred.
Example 1: “DOMINIQUE” -unisex name resolved by Tier 0
Dictionaries say “Dominique” is used for both sexes. But the administration addressed this person as “Cher Monsieur” — direct evidence of how they perceived the requester. Since our study examines the effect of perceived gender on responsiveness, this perception is the relevant signal.

Tier 0 takes precedence because it reflects the administration’s direct perception of the requester’s gender — the signal that drives the bureaucratic behavior under study. Tiers 1–3 still run for audit purposes. Tier 0 resolved 1,339 requester cases (77 with no name, 74 unisex, 23 disagreements with dictionaries). Tier 4 (LLM) uses temperature=0 and seed=42 for near-deterministic results; all its M/F outputs are flagged for manual review.
Step 10 of 17
Tier 4: The LLM Fallback
Only cases unresolved by Tiers 0–3 reach the LLM. With Tier 0 resolving many requester cases via response greetings, fewer cases need LLM classification. An anti-hallucination prompt prevents the model from guessing.
SYSTEM PROMPT (requester): You are classifying the sex of the person who signed a FOI request. Look for a PERSONAL SIGNATURE after the closing formula. IMPORTANT: Many FOI requests have NO personal signature - they may be signed by organisations, pseudonyms, or contain only initials. If you do not see a clear first name + surname, return UNKNOWN. Do NOT guess or invent a name. Examples of valid: “Jean Dupont” → M, “Marie Lambert” → F Examples of UNKNOWN: “N. Dupont”, “Collectif du Bempt”, “Service Juridique” Reply EXACTLY: GENDER: <M or F or UNKNOWN> | NAME: <first name or NONE>
Two Safety Filters
Filter 1 - False positive filter: If the LLM returns a common French function word as a "name" (e.g., "Le", "Je", "Madame", "Merci"), the result is silently discarded. These words are checked against a blocklist of articles, pronouns, prepositions, and common greetings.

Filter 2 - Hallucination filter:
1. LLM returns: GENDER: M | NAME: Philippe
2. Filter checks: does "Philippe" appear in the input text?
3. If YES → accept classification, flag for manual review
4. If NO → hallucination detected, reclassify as UNKNOWN
Filter Results (All Tier 4 — Requester + Respondent Combined)
Before filtering: 140 LLM M/F responses
False positive filter: 0 caught (Qwen never returned function words as names)
Hallucination filter: 14 caught (names not in text, e.g., fabricated names)
After filtering: 126 clean M/F accepted (10.0% rejection rate)

Step 13 of 17
Respondent Processing
The respondent lane follows the same steps but with extra cleaning: quoted original messages must be stripped first.
Madame, Monsieur, Nous accusons réception de votre demande d'accès du 14/03/2022. Votre demande a été transmise au service compétent. Le Secrétariat communal Commune de Rebecq secretariat@rebecq.be -----Message d'origine----- De: transparencia@example.be Envoyé: lundi 14 mars 2022 14:30 Objet: Demande d'accès - permis d'urbanisme [original request text quoted here...]
Key challenge: The signer here is “Le Secrétariat communal -Commune de Rebecq” -an institutional signature with no personal name. spaCy NER finds no PERSON entity. Heuristic finds no known first name. Result: UNKNOWN.
Quoted message removal is critical: without it, the pipeline might extract the requester's name from the quoted text instead of the respondent's name. The pipeline detects headers like “-----Message d'origine-----”, “Van: ... Verzonden:” (NL), “De: ... Envoyé:” (FR).

Step 15 of 17
Validation: Confusion Matrix
Three methods compared: the old ChatGPT approach, the initial Llama pipeline, and the final Qwen+Tier 0 pipeline. Higher agreement with ChatGPT = more consistent. More known-sex pairs = more usable data.
Pipeline evolution: The pipeline evolved through two iterations. An initial run with Llama 3.2 (3B parameters) produced a 74% error rate in Tier 4 classifications, identified during manual review by a co-author. The pipeline was re-run with Qwen 2.5 (7.6B) and Tier 0 (response greeting detection, suggested by a co-author). The table below compares both iterations against the original ChatGPT classifications.

Why only Tier 4 M/F cases and Tier 0 disagreements? Tiers 1–3 are fully deterministic dictionary lookups with curated, published sources. The LLM (Tier 4) is probabilistic and can hallucinate -so every M/F classification it makes is verified by a human researcher. Tier 0 (greeting) disagreements with name-based tiers are also reviewed before inclusion in the final dataset.
What about the UNKNOWN cases?
Cases where Tier 4 returned UNKNOWN were not included in the M/F review because no gender claim was made. To verify this was justified, a 200-sample manual review of Qwen UNKNOWN outputs was conducted by reading the original FOI texts.
