🔍 Image-to-Blender Detection Agent

CV-based element detection from a GUI PNG → 3D Blender scene with each element as a separate mesh

What it does

Takes any VST GUI PNG (output from ChatGPT, Gemini, a real plugin screenshot, or a decomposition sheet) and auto-detects the interactive elements — knobs (big + small), rectangular controls (sliders, buttons, piano keys), keyboard strips — by layered OpenCV analysis. Exports a single JSON spec which a Blender script consumes to build each element as a separate, precisely-positioned 3D object aligned with the chassis PNG underneath.

Why this matters

Previous attempts (freehand sketch-over-image, Blender-only procedural, AI image→3D APIs) all failed for different reasons: imprecise coords, hallucinated controls, single merged meshes. This agent solves the coordinate accuracy problem deterministically using computer vision, and leaves creative/style work to the upstream GUI-design prompts.

Files

  • detect.py — OpenCV detection (Hough circles + contours + keyboard strip detection). Outputs a JSON spec + debug PNG overlay.
  • apply.py — Blender-side builder. apply_spec('/path/to/spec.json', chassis_png='/path/to/chassis.png') sets up an image-plane chassis and builds all detected elements as separate meshes on top.

Detection stages

  1. Hough circles — knob band (r 22-55px): main rotary controls
  2. Hough circles — small band (r 8-22px): EQ nodes, tiny buttons (excludes positions near bigger knobs to avoid cap double-detection)
  3. Canny + contours: rectangular controls classified by aspect ratio into slider / horizontal button / square button candidates
  4. Brightness-threshold strip: detects the bottom keyboard region and its bounds

Usage

# 1. Detect
python3 detect.py path/to/chassis.png --out path/to/spec.json

# 2. Apply in Blender (scripting tab or via mcp-blender)
exec(open('apply.py').read())
apply_spec('path/to/spec.json', chassis_png='path/to/chassis.png')

Output format

{
  "image": "chassis.png", "width": 1619, "height": 971,
  "knobs":         [{"cx": 176, "cy": 294, "r": 49}, ...],
  "small_circles": [{"cx": 1512, "cy": 792, "r": 28}, ...],
  "rectangles":    [{"x": 218, "y": 270, "w": 40, "h": 250, "kind": "vertical_slider_candidate"}, ...],
  "keyboard_strip":{"x1": 40, "y1": 810, "x2": 1250, "y2": 955}
}

Position in the full pipeline

  1. Moodboard Generator — extract style DNA (optional)
  2. VST GUI Pipeline — ChatGPT generates the full redesigned GUI PNG
  3. THIS AGENT — analyze the PNG, extract element positions as JSON
  4. Blender Redesign / Blender ops — build the 3D scene from the JSON
  5. GUI Frame Tool — manual fine-tuning of layout if needed

Tunable parameters

In detect.py:

  • Knob band: minRadius=22, maxRadius=55 — widen if your knobs are larger
  • Hough sensitivity: param2 (lower = more false positives, higher = fewer detections)
  • Header/footer exclusion: 70 < cy < 810 — adjust for your chassis proportions
  • Keyboard strip brightness threshold: 170 — raise for darker keyboard beds

Known limitations

  • Decorative circles (logo rings, ornaments) can leak into the knob set — prune manually or add stricter size filters
  • Contour detection over-produces rectangles on heavily-textured chassis; may need a masked ROI pass
  • Piano key count is assumed 14 white — override num_white in apply.py for different ranges
  • Does not yet handle: arc segments, concentric ring structures, text labels, color-coded states
