🎛️ VST GUI Design Pipeline

3-prompt ChatGPT workflow: style transfer → element redesign → decomposition

What it does

A structured workflow for redesigning a VST plugin GUI using ChatGPT (or Gemini). Three sequential prompts, all meant to run in the same conversation so the style DNA from the first prompt propagates to everything downstream.

The 3 prompts

1️⃣ Full GUI generation

01_full_gui_prompt.txt

Input: (1) screenshot of your current plugin in a DAW (functional layout only — style is rough) + (2+) screenshots of VSTs whose aesthetic you want to borrow. Output: a full plugin window in the reference style with YOUR layout preserved.

2️⃣ Hero element redesign (novel geometry)

02_element_redesign_prompt.txt

After step 1 succeeds, use this to get a SINGLE signature control radically reinvented in the same style. Explicitly forbids the "concentric rings" default and prompts ChatGPT with concrete creative directions (compass, stained-glass, planetary, ritual artifact, chord-wheel, etc.).

Input: (1) full GUI from step 1 + (2) your rough concept diagram of the element (for parameter list only, NOT geometry).

3️⃣ Full decomposition

03_decompose_prompt.txt

Input: full GUI from step 1. Output: reference sheet PNG with every interactive element isolated at consistent scale on a neutral background — used to rebuild each piece cleanly in Blender with separate meshes.

Workflow

Open ChatGPT, new chat
─────────────────────────
Msg 1: upload Ableton screenshot + style refs → paste 01 prompt → receive full GUI
Msg 2: upload full GUI + element concept      → paste 02 prompt → receive novel hero
(repeat Msg 2 for other elements as needed)
Msg 3: upload full GUI                        → paste 03 prompt → receive decomposition
─────────────────────────
Send full GUI + hero + decomposition to Claude for Blender rebuild

Why this beats a single prompt

  • Split attention: ChatGPT hallucinates when asked to juggle "generate GUI + coord table + decomposition" in one turn.
  • Style carries through the conversation: output 1 becomes style reference for output 2 and 3.
  • Hero element gets its own dedicated prompt with anti-default instructions. Without this, ChatGPT always falls back to concentric rings — the opposite of "hero design."
  • Decomposition preserves element boundaries: isolated elements at consistent scale → clean inputs for Blender separate-mesh rebuild.

When this is useful

  • Any VST GUI redesign where you have a working layout but want a fresh aesthetic.
  • When Blender procedural shaders hit their quality ceiling and you want photoreal-mood reference.
  • As design-direction exploration before committing to a specific visual language.

Known failure modes + fixes

  • Gemini returns IMAGE_FAILED: quota issue. Wait 30-60 min or switch to ChatGPT.
  • Style transfer weak: upload 3-4 style refs instead of 1. Model averages the signal.
  • Default concentric rings: use Prompt 02, it forbids them explicitly.
  • Hallucinated controls: restate "use ONLY what's in IMAGE 1" at the top of the prompt.
